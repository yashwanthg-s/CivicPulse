## ✅ Checklist

### Before You Start
- [ ] Read one of the overview documents
- [ ] Understand the two issues (resolution images, category storage)
- [ ] Have backend and frontend ready to run

### Resolution Images Setup
- [ ] Run migration: `node backend/run-migration.js`
- [ ] Start backend: `npm run dev`
- [ ] Start frontend: `npm run dev`
- [ ] Test through UI
- [ ] Verify database and files

### Category Storage Fix
- [ ] Open `backend/controllers/complaintController.js`
- [ ] Find line: `const aiCategory = geminiResponse.category || category || 'other';`
- [ ] Change to: `const aiCategory = category || geminiResponse.category || 'other';`
- [ ] Test by submitting complaint with category
- [ ] Verify database has category

### Verification
- [ ] Backend console shows detailed logs
- [ ] Images saved to `backend/uploads/`
- [ ] Database has resolution records
- [ ] Citizens see before/after images
- [ ] Admins see resolutions
- [ ] Category is stored correctly
